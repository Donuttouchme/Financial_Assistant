# Post-v1.2.1 Bug Batch Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix the 21 distinct bugs surfaced by the post-v1.2.1 code review, ship two stalled features (full recurring-schedule management, full backup/restore), and close gaps that produce stale UI state, broken CSV exports, and silent forecast misbehaviour.

**Architecture:**
- Backend: extend `app/migrations.py` with a single nullable `transactions.schedule_id` column; add `recurring.py` and `backup.py` routers; flesh out `backup_service` with pre-restore archives + atomic swap; surgical fixes inside existing services for FX weekend lookup, Jan-31 drift, forecast window alignment.
- Frontend: add forecast cache invalidations to the three mutation hooks that miss them, wire `commitImport` through TanStack Query, add emerald + navy to `AppearanceSection`, build a new `/recurring` management page and a backup section under Settings, top-level `ErrorBoundary`, plug missing MSW handlers so future regressions don't slip through.
- Migration strategy: nullable column add via the existing idempotent `migrations.py`. ON-DELETE-SET-NULL is enforced at the SQLAlchemy session level (SQLite ALTER TABLE cannot add FK constraints).

**Tech Stack:** Python 3.14, FastAPI, SQLAlchemy 2.0, pydantic v2, SQLite + custom `migrations.py` (no Alembic). React 18 + TS strict, TanStack Query v5, Tailwind 3 + shadcn-ui, Recharts, Vitest 2 + MSW 2, react-router-dom 6.

```tsx
import React from "react";
import { Button } from "@/components/ui/button";

interface State { hasError: boolean; }

export class ErrorBoundary extends React.Component<{ children: React.ReactNode }, State> {
  state: State = { hasError: false };
  static getDerivedStateFromError() { return { hasError: true }; }
  componentDidCatch(error: unknown, info: unknown) {
    console.error("ErrorBoundary caught", error, info);
  }
  render() {
    if (!this.state.hasError) return this.props.children;
    return (
      <div className="flex h-screen items-center justify-center p-8">
        <div className="max-w-md space-y-4 text-center">
          <h1 className="text-xl font-semibold">Something went wrong</h1>
          <p className="text-muted-foreground">
            The page hit an unexpected error. Reloading usually fixes it.
          </p>
          <Button onClick={() => window.location.reload()}>Reload</Button>
        </div>
      </div>
    );
  }
}
```

```ts
const HORIZONS: ForecastHorizon[] = ["1m", "3m", "6m", "1y", "2y"];
const MODES: ForecastMode[] = ["centered", "forward"];

export function parseHorizon(raw: string | null | undefined): ForecastHorizon {

}

export function parseMode(raw: string | null | undefined): ForecastMode {
  return MODES.includes(raw as ForecastMode) ? (raw as ForecastMode) : "centered";
}

export function parseCategoryId(raw: string | null | undefined): number | undefined {
  if (!raw) return undefined;
  const n = Number(raw);
  return Number.isInteger(n) && n > 0 ? n : undefined;
}
```

```python
import shutil
from datetime import datetime
from pathlib import Path
from app.config import settings

_PRE_RESTORE_KEEP = 5

def export_db() -> bytes:
    return Path(settings.resolved_db_path()).read_bytes()

def stage_restore(uploaded: bytes) -> Path:
    db = Path(settings.resolved_db_path())
    pending = db.parent / f"{db.name}.pending"
    pending.write_bytes(uploaded)
    validate_uploaded_db(pending)
    return pending

def commit_restore() -> Path:
    db = Path(settings.resolved_db_path())
    pending = db.parent / f"{db.name}.pending"
    assert pending.exists(), "no pending restore"
    stamp = datetime.now().strftime("%Y%m%d-%H%M%S")
    archive = db.parent / f"{db.name}.pre-restore-{stamp}"
    if db.exists():
        shutil.move(db, archive)
    shutil.move(pending, db)
    _prune_archives(db)
    return archive

def _prune_archives(db: Path) -> None:
    archives = sorted(db.parent.glob(f"{db.name}.pre-restore-*"))
    for old in archives[:-_PRE_RESTORE_KEEP]:
        old.unlink()
```

## Notes for the implementer

- **Windows shell**: PowerShell — chain with `;`, not `&&`. Use `& "./.venv/Scripts/python.exe"` to call the venv Python.
- **`base_amount`** is computed on read in `enrich_with_base_amount`, never stored. Don't try to backfill anything.
- **FX storage convention**: `FxRate.rate_to_eur` stores frankfurter raw (`1 EUR = N currency`). Consumers convert as `amount × rate(base) / rate(native)`. This was flipped in v1.2.1 — don't re-flip.
- **`Decimal` everywhere** on the backend; never floats. JSON responses serialise to string by default.
- **EUR is the pivot** — same-currency short-circuits before any FX lookup. Tests that need clean math use `set_base_currency(db, "EUR")`.
- **No Alembic** — column adds go in `app/migrations.py` as nullable. Anything more complex needs human review.
- **Plan deviations are fine** if you discover the spec is wrong. Annotate the plan with a ⚠️ block describing what changed and why. (This happened in the v1.2 plan and was the right call.)
